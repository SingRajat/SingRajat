## About

I build **production-grade AI systems** — not demos, not throwaway scripts. My work lives at the intersection of LLM orchestration, agentic architecture, and backend observability. I care about how AI behaves under real load: retrieval precision, latency distributions, failure modes, and traceability.

Engineering philosophy: *instrument before you optimize — understand before you claim.*

- I architect **context-aware RAG systems** with hybrid retrieval that go beyond naive vector search
- I design **agentic pipelines** with LangGraph and CrewAI that operate reliably across multi-turn, stateful workloads  
- I ship with **Prometheus + Grafana** from day one — observability is part of the design, not an afterthought
- I write **FastAPI backends** built for concurrency, session integrity, and clean separation of concerns

```
Design → Instrument → Build → Observe → Harden → Scale
```

## Architecture Mindset

```
Problem Framing     →  Constraints first. Abstractions second.
   ↓
System Design       →  API contracts · data models · failure modes · SLOs
   ↓
Implementation      →  Clean, typed, async-safe Python
   ↓
Instrumentation     →  Metrics and traces before first deploy
   ↓
Production          →  Hardened with observability, not hope
   ↓
Iteration           →  Data-driven feedback loops, not gut-driven rewrites
```

For AI systems specifically:

- **Retrieval** must be measurable — RRF scores, re-ranking deltas, not vibes
- **Agents** must be traceable — every tool call, every state transition, logged
- **Generation** must be grounded — source-cited, hallucination-bounded
- **Latency** is a product requirement, measured at p50/p95/p99
